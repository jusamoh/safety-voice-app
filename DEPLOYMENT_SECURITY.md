# 배포 전 로그인 환경변수 설정

이 변경은 저장소에 있던 로그인 정보를 제거하고 Render 환경변수에서만 읽도록 바꿉니다. **아래 설정을 완료하기 전에 PR을 병합하면 로그인이 503 응답으로 차단됩니다.**

## Render 설정

1. Render Dashboard에서 이 서비스를 선택합니다.
2. **Environment** 메뉴에서 `APP_USERS_JSON` 환경변수를 추가합니다.
3. 값에는 사용자명과 새 비밀번호를 JSON 객체로 입력합니다.

```json
{"admin":"새로운-긴-비밀번호","operator":"또-다른-긴-비밀번호"}
```

4. 기존 저장소에 노출된 비밀번호는 모두 폐기하고, 각 계정마다 서로 다른 16자 이상의 새 비밀번호를 사용합니다.
5. 저장 후 서비스를 재배포하고 `/api/login` 로그인을 확인합니다.
6. 로그인이 확인된 다음 이 PR을 병합합니다.

## 동작 방식

- `APP_USERS_JSON`이 없거나 JSON 형식이 잘못되면 서버는 로그인 요청에 `503`을 반환합니다.
- 인증 실패는 기존과 동일하게 `401`을 반환합니다.
- 서버가 재시작되면 메모리의 로그인 토큰이 초기화되므로 사용자는 다시 로그인해야 합니다.

## PWA 및 다국어 UI 배포 확인

PWA와 한국어·영어·일본어·중국어 UI는 별도의 Render 환경변수를 요구하지 않습니다. 기존 빌드 및 실행 명령을 그대로 사용합니다.

배포 저장소에는 아래 파일과 디렉터리가 함께 포함되어야 합니다.

- `i18n.js`
- `manifest.webmanifest`
- `service-worker.js`
- `icons/icon-192.png`
- `icons/icon-512.png`
- `icons/icon-maskable-512.png`

배포 후 다음 주소가 모두 `200`을 반환하는지 확인합니다.

- `/i18n.js`
- `/manifest.webmanifest`
- `/service-worker.js`
- `/icons/icon-192.png`

브라우저에서 관리자 언어를 한국어, 영어, 일본어, 중국어로 각각 변경해 메뉴와 경고 문구가 함께 전환되는지 확인합니다. PWA 서비스 워커는 정적 화면 파일만 캐시하며 `/api/`, `/ws`, `room_token`이 포함된 주소는 캐시하지 않습니다.
